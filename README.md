# newEarth

> Try to build a new Earth — 一个由 LLM Agent 共同生活、工作、繁衍的 2D 数字平行世界。

## 项目简介

**newEarth** 是一个受 Stanford Generative Agents（Smallville）启发的多人分布式 Agent 社会模拟项目。与原项目不同的是，newEarth 的核心设定是**"玩家共建 + 自付算力 + 真实经济闭环"**：

- 🧑‍🤝‍🧑 **分布式共建**：每个用户创建自己的角色（人设、职业、背景），加入共享的 2D 世界
- 🔥 **自付算力**：角色的"思考"消耗用户自己的 LLM token（BYOK，Bring Your Own Key）
- 👁️ **观察与介入**：用户可以实时观察自己角色的生活，在关键节点通过 prompt 介入其决策
- 💰 **世界经济**：世界发行数字货币，角色通过工作、交易、投资赚取世界币
- 🛒 **商店与充值**：用户可用现实货币充值获得世界币，购买虚拟资产（土地、房屋、道具）
- 🌉 **真实世界桥接**（长期目标）：角色可作为"数字劳动力"对外提供真实服务（编程、设计、写作），赚取真实收入反哺世界

### 核心价值主张

1. **涌现式社会**：不预设剧本，社会关系、经济秩序、文化习俗由 Agent 互动涌现
2. **分层思考架构**：平台提供"基础意识"（保底），用户 token 驱动"高级思考"（上限）
3. **养成 × 观察 × 投资**：集电子宠物、沙盒模拟、经济游戏于一体的新品类

### 合规边界（重要）

> 为规避金融监管风险，PoC 阶段采用**单向资金流**设计：
> - ✅ 现金 → 世界币（游戏内充值，同《原神》《魔兽世界》性质）
> - ✅ Agent 对外服务 → 现金（劳务收入，不经过世界币兑换）
> - ❌ 世界币 → 现金（不支持提现，避免证券化/赌博认定）

```
   ┌─────┴─────┐      ┌─────┴─────┐       ┌─────┴─────┐

   │ 烧 A 的key│      │ 烧 B 的key│       │ 平台兜底   │
   └───────────┘      └───────────┘       └───────────┘
```

### 分层思考架构

| 层级 | 负责 | 算力提供方 | 模型建议 |
|------|------|------------|----------|
| **L1 基础意识** | 走路、吃饭、状态机、基础反应 | 平台 | Haiku / 规则引擎 |
| **L2 日常思考** | 对话、反思、规划、创作 | 用户自付 | Sonnet |
| **L3 关键决策** | 人生节点、重大选择（用户可介入） | 用户自付 | Opus |

## PoC 开发计划

PoC 目标：在**最小闭环**内验证"分布式 Agent 共建 + token 经济"的可行性。**暂不包含**繁衍、法律体系、外接真实服务、现金兑换。

### 里程碑总览

| 阶段 | 代号 | 目标 | 交付物 |
|------|------|------|--------|
| M0 | Foundation | 项目脚手架与世界骨架 | 可运行的空世界 |
| M1 | Single Agent | 单 Agent 生命循环 | 一个会生活的 NPC |
| M2 | World Loop | 多 Agent 共存与事件系统 | 5 个 NPC 共同生活 |
| M3 | Economy | 经济系统与职业 | 能工作赚币的 Agent |
| M4 | Multiplayer | 用户账号与角色创建 | 玩家可加入世界 |
| M5 | BYOK | 用户自付 token 接入 | 完整经济闭环 Demo |

---

### M0 · Foundation（脚手架）

**目标**：搭好项目骨架，跑通一张空地图。

- [ ] 仓库目录结构（monorepo: `server/` `client/` `shared/`）
- [ ] 后端：FastAPI + PostgreSQL + Redis
- [ ] 前端：Vite + React + PixiJS（2D 渲染）
- [ ] WebSocket 双向通信骨架
- [ ] 世界时钟（Tick 系统，支持暂停/加速）
- [ ] 一张 50×50 的瓦片地图（静态配置）
- [ ] 开发环境 Docker Compose 一键启动

**验收**：浏览器打开能看到地图，时间在走，前后端通信正常。

---

### M1 · Single Agent（单体生命）

**目标**：让一个 Agent 在世界里"活"起来。

- [ ] Agent 数据模型（身份、属性、位置、状态、记忆）
- [ ] L1 基础行为循环（FSM：idle / move / eat / sleep / work）
- [ ] 记忆系统 v1：memory stream（事件流 + 时间衰减）
- [ ] L2 思考接入：每 N 个 tick 调用 LLM 生成日程/反思
- [ ] Agent 渲染：精灵、移动动画、状态气泡
- [ ] 调试面板：点击 Agent 查看其记忆、当前目标、思考日志

**验收**：地图上有一个小人会自己起床、走路、吃饭、睡觉，点开能看到它"在想什么"。

---

### M2 · World Loop（多体共存）

**目标**：多个 Agent 互相感知、交互。

- [ ] 感知系统（视野范围内的其他 Agent 与物体）
- [ ] 对话系统：相遇触发对话，LLM 生成内容
- [ ] 事件总线：世界事件广播（天气、公告、突发事件）
- [ ] 反思机制：Agent 定期基于记忆生成 high-level insight
- [ ] 5 个预置 NPC，不同性格与日程
- [ ] 时间加速：支持 1x / 10x / 60x 世界速度

**验收**：5 个 NPC 能自发相遇、对话、形成初步关系，对话历史可回放。

---

### M3 · Economy（经济与职业）

**目标**：引入世界币与基础经济活动。

- [ ] 世界币账本（双记账，不可篡改的交易日志）
- [ ] 职业系统 v1：程序员、厨师、商人、艺术家、农夫
- [ ] 工作地点与产出机制（"上班" → 消耗时间 + token → 产出世界币）
- [ ] 商店系统：NPC 商店（买食物、工具、衣服）
- [ ] 房屋与租赁：基础虚拟地产
- [ ] 经济面板：查看个人资产、全局经济指标

**验收**：NPC 能自主选择职业、上下班、消费、存钱，经济循环自洽运行 1 个虚拟周不崩盘。

---

### M4 · Multiplayer（玩家接入）

**目标**：真实用户能创建角色加入世界。

- [ ] 用户账号系统（注册/登录/JWT）
- [ ] 角色创建向导（姓名、外观、性格、职业、背景故事）
- [ ] 角色观察视图（第三人称跟随自己的角色）
- [ ] 关键节点介入：当 Agent 触发"重大决策"时推送通知给用户，用户可 prompt 建议
- [ ] 每日介入次数限制（防止用户手操变木偶）
- [ ] 角色间互动（用户 A 的角色可与用户 B 的角色对话）

**验收**：两个用户分别创建角色，能在同一世界里相遇、交流、观察各自的生活。

---

### M5 · BYOK（自付算力）

**目标**：打通"用户 token → 世界币"的核心经济循环。

- [ ] 用户 API Key 加密存储（Anthropic Key）
- [ ] Token 计量：每次 LLM 调用记录消耗
- [ ] "思考即劳动"：Agent 思考消耗的 token 折算为生产力加成
- [ ] 商店充值模拟：沙盒支付（不接真实支付通道）
- [ ] 离线模式：用户离线时 Agent 降级到 L1 平台算力运行
- [ ] 数据面板：个人 token 消耗、世界币余额、资产净值

**验收**：用户使用自己的 Anthropic Key 驱动角色思考，角色通过工作赚取世界币，用赚到的币 + 沙盒充值购买虚拟资产。

## PoC 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 后端 | Python 3.11 + FastAPI | 生态好，LLM 集成方便 |
| 数据库 | PostgreSQL + Redis | 账本强一致 + 实时状态缓存 |
| 向量存储 | pgvector | 简化部署，不引入第二个数据库 |
| 前端 | React + Vite + PixiJS | Web 上手快，2D 性能够用 |
| 实时通信 | WebSocket | 世界事件推送 |
| LLM | Anthropic Claude (Haiku/Sonnet/Opus) | 分层调用，Haiku 兜底 |
| 部署 | Docker Compose（本地） | PoC 阶段不上云 |

## PoC 阶段不做的事

明确排除，避免范围蔓延：

- ❌ 繁衍 / 基因 / 家族谱系
- ❌ 法律 / 议会 / 立法系统
- ❌ Agent 对接真实客户提供服务
- ❌ 真实支付通道与世界币兑现
- ❌ 区块链 / NFT / 链上资产
- ❌ 移动端客户端
- ❌ 多服务器 / 大规模分布式部署

这些会在 PoC 验证通过后进入 **v1.0 路线图**。
